# hc_qa/ Folder Structure — v1 (2026-07-21)

Master flow chart of the whole QA test tree. One folder per application under `hc_qa/api/`
(backend APIs) and `hc_qa/web/` (web/mobile apps) — see `hc_agile/team/dev_qa_agent.md` for the
ownership convention.

```
hc_qa/
├── api/
│   ├── ai_hc_api/                       ← AI.HealthCare.Patient.API
│   │   ├── tests/patients.spec.ts         (17 tests)
│   │   ├── fixtures/patient.ts
│   │   └── demo/                          (15 resource upsert/import scaffolds)
│   │
│   ├── hc_ai_identity_api/              ← HC.AI.Identity.Api (auth backend)
│   │   ├── tests/users.spec.ts            (19 tests)
│   │   └── fixtures/user.ts
│   │
│   ├── hc_ai_mapi/                      ← HC.AI.MAPI — skeleton, no tests yet
│   └── hc_fapi/                         ← hc_fapps (FastAPI) — skeleton, no tests yet
│
└── web/
    ├── aihcweb/                         ← Angular chat/auth/admin UI
    │   ├── demos/
    │   │   ├── demo1/  (login → Doctor Chat)
    │   │   ├── demo2/  (login → Patient Chat)
    │   │   └── demo3/  (Signup walkthrough)
    │   ├── tests/
    │   │   ├── login/login.spec.ts          (6 tests)
    │   │   ├── signup/signup.spec.ts        (8 tests)
    │   │   └── admin-login/admin-login.spec.ts (6 tests)
    │   └── fixtures/
    │       ├── auth.ts        ← real-API users (buildTestUser/seedUser)
    │       └── admin-auth.ts  ← mock-only admin users (buildAdminTestUser/seedAdminAccount)
    │
    └── mvkhcapp/                        ← skeleton, no tests yet
```

## Total test count as of this version
- Playwright: 17 + 19 + 6 + 8 + 6 = **56 tests**
- Backend xUnit (audited, not QA-authored): 7 + 1 + 1(scaffold) + 11 = 20 test methods across 4 projects
- Angular unit (Dev Angular-authored, QA-validated not QA-authored): 58 `it()` blocks
