# hc_qa/ Folder Structure — v2 (2026-07-21)

Supersedes [v1](v1_2026-07-21.md). `hc_qa/web/aihcweb/tests/` restructured from flat
`login/`/`signup/`/`admin-login/` folders into a user-type hierarchy — `admin-user/` vs.
`site-user/`, with `site-user/` further split into `doctor-user/`/`patient-user/`. This is ahead of
any actual need: `/login` and `/signup` are still one shared page for Doctor and Patient today,
so `doctor-user/` and `patient-user/` currently hold near-duplicate test content — done deliberately,
to have the segregation ready before validation diverges, rather than after.

## What changed vs. v1
| v1 path | v2 path |
|---|---|
| `tests/login/login.spec.ts` | `tests/site-user/doctor-user/login/login.spec.ts` **and** `tests/site-user/patient-user/login/login.spec.ts` (duplicated) |
| `tests/signup/signup.spec.ts` | `tests/site-user/doctor-user/signup/signup.spec.ts` **and** `tests/site-user/patient-user/signup/signup.spec.ts` (duplicated, persona-adjusted) |
| `tests/admin-login/admin-login.spec.ts` | `tests/admin-user/admin-login/admin-login.spec.ts` (moved, not duplicated) |
| *(none)* | `tests/admin-user/admin-signup/admin-signup.spec.ts` — new, 5 cases |

Doctor's signup suite swaps the original Patient-specific "lands on Patient home view" case for a
symmetric "lands on Doctor Chat view" case; Patient's suite keeps the original. Both otherwise
carry the same generic validation cases (mismatched passwords, short password, missing fields,
whitespace name, invalid email format, duplicate-email-upserts) — currently identical between the
two since the underlying page is shared.

## Known cost of this decision
Duplicated content means a bug fix in shared login/signup behavior needs applying in both
`doctor-user/` and `patient-user/` copies until they actually diverge. Accepted deliberately — see
`hc_agile/scrum/tasks/QA-002_US009_aihcweb_auth_ui_test_cases.md`.

## New test count
Playwright: 17 (`ai_hc_api`) + 19 (`hc_ai_identity_api`) + 6+8 (doctor-user) + 6+8 (patient-user) +
6 (admin-login) + 5 (admin-signup) = **75 tests** (up from 56 in v1).
