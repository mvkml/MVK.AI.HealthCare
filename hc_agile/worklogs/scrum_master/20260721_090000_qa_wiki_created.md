## Subject: QA wiki created — module-wise, version-wise archive

## What Happened
Created `hc_agile/wiki/` — a local, version-controlled archive of QA folder structure and test
coverage documentation, organized one folder per module with dated version files
(`v<N>_<YYYYMMDD>.md`), so history is preserved rather than overwritten. Mirrors the pattern
already used for the Azure DevOps Wiki's LLM section (`hc_agile/hc_llm/ollama/`).

## Action
Created `v1_2026-07-21.md` for 7 modules:
- `qa/hc_qa_folder_structure/` — the overall `hc_qa/` tree
- `qa/login/`, `qa/signup/`, `qa/admin-login/` — per-page Playwright coverage
- `qa/identity-api/`, `qa/ai-hc-api/` — per-API Playwright coverage
- `qa/epics-features/` — Epic → Feature → User Story hierarchy snapshot

Each version file captures a text flow chart of the relevant folder plus a test-case table with
current pass/fail status, and links back to the live QA-xxx/USxxx tracking files (this wiki is an
archive/snapshot layer, not a replacement for those).
